Notes

We used GOV.UK's button as a starting point https://design-system.service.gov.uk/components/button/ - but due to a number of NHS.uk's elements having squared edges we didn't want the buttons to get lost, so we added 4px radius (rounded corners) to add extra affordance (users will know its a button, not a heading or panel).

How it works

Using GDS advice:

  • Write button text in sentence case, describing the action it performs. For example ‘Save and continue’ or ‘Start now’.
  • Align the primary action button to the left edge of your form.

Our hypothesis

We believe

  • gov buttons are well established (and tested)
  • gov button's squared corners lack affordances within the NHS design system - so need to look more 'button-y'
  • Green is a good colour for primary actions
  • Its best to stay clear of blue as a button colour due to the amount of blue within the NHS Design system

Accessibility

Our accessible considerations are base on simple principles

  • Ensure that the words used describe the action it performs (and that it will make sense when read out by a screen reader)
  • Colour contrast meets WCAG AAA standard (primary button has a contrast ratio of 4.93, secondary 7.75. reverse 14.42)

Usage

Going forward

Tables will need captions. Currently on NHS.uk's wagtail we don't have the option to include captions so this will need to be explored further.

We'd also like to look at tables with more than three columns, tables that are mostly data and tables that could be reordered by a user.

What is success

Success would be a user being able to find the information they require.

How we measure success

Success can be measured by page depth, exit rate and page depth and also monitoring a feedback survey to find if any users mentioned not being able to find information that is presented in the table.
